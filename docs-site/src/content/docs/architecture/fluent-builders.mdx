The fluent builders are Layer 1 of the architecture — the developer's API surface. Every method call creates a **descriptor** (a data structure), not a side effect. The plan collects descriptors. `Render()` serializes them to JSON. The runtime executes the JSON.

```d2
direction: right

call: "Developer writes\nfluent C# in .cshtml" {
  shape: rectangle
  style.fill: "#e8f0fe"
  style.stroke: "#4285f4"
  style.font-size: 18
}
descriptor: "Descriptors created\nTrigger, Reaction,\nCommand, Mutation" {
  shape: rectangle
  style.fill: "#fef7e0"
  style.stroke: "#f9ab00"
  style.font-size: 18
}
plan: "Plan collects\nEntries + ComponentsMap" {
  shape: rectangle
  style.fill: "#e6f4ea"
  style.stroke: "#34a853"
  style.font-size: 18
}
json: "plan.Render()\nJSON in the page" {
  shape: rectangle
  style.fill: "#d4edda"
  style.stroke: "#34a853"
  style.font-size: 18
}

call -> descriptor: builds {
  style.stroke: "#4285f4"
}
descriptor -> plan: collects {
  style.stroke: "#f9ab00"
}
plan -> json: serializes {
  style.stroke: "#34a853"
}
```

## The boundary rule

The C# side builds descriptors. It never touches the DOM, never calls `fetch`, never evaluates conditions. The JS side reads the plan and executes it. It never creates new commands, never extends the schema, never invents behavior.

If a feature can't be expressed as a plan descriptor, it doesn't exist in the framework. The plan is the only contract.
