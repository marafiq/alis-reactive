## What It Looks Like

<CardGrid>
	<Card title="Cascading Dropdowns" icon="random">
		Pick a facility — units populate automatically. Two parallel GETs, zero JavaScript.
		```csharp
		pipeline.Get("/Intake/Units")
		    .Gather(g => g.Include<FusionDropDownList,
		        Model>(m => m.FacilityId))
		    .Response(r => r.OnSuccess<UnitsResponse>(
		        (json, s) => s.Component<FusionDropDownList>(
		            m => m.UnitId)
		            .SetDataSource(json, j => j.Units)
		            .DataBind()));
		```
		[See full example](/examples/resident-intake/)
	</Card>
	<Card title="Conditional Visibility" icon="approve-check">
		Check a box — a section appears. Pick a value — another section appears. All declarative.
		```csharp
		pipeline.When(args, a => a.Checked).Truthy()
		    .Then(t => t.Element("section").Show())
		    .Else(e => e.Element("section").Hide());
		```
		[Conditions reference](/csharp-modules/reactivity/conditions/)
	</Card>
	<Card title="HTTP + Validation + Toast" icon="rocket">
		One pipeline: confirm, validate, POST, loading overlay, toast, chained follow-up request.
		```csharp
		pipeline.Confirm("Submit?")
		    .Then(t => t
		        .Post("/save", g => g.IncludeAll())
		        .Validate<MyValidator>("form")
		        .Response(r => r.OnSuccess(s =>
		            s.Component<FusionToast>()
		                .SetContent("Saved!")
		                .Success().Show())));
		```
		[HTTP pipeline](/csharp-modules/reactivity/http-pipeline/)
	</Card>
	<Card title="Dynamic Partials" icon="puzzle">
		Load a server-rendered partial with input fields via `Into()`. Plan merging enriches validation automatically.
		```csharp
		// Partial uses ResolvePlan — merges into parent
		var plan = Html.ResolvePlan<Model>();
		Html.InputField(plan, m => m.Room,
		    o => o.Required().Label("Room"))
		    .NativeTextBox(b => b.Placeholder("..."));
		```
		[Worked example](/examples/resident-intake/)
	</Card>
</CardGrid>

## Core Principles

<CardGrid>
	<Card title="Plan Is the Contract" icon="document">
		The JSON plan is the **only** contract between C# and JS. No shortcuts, no manual JS in views.
	</Card>
	<Card title="Three Test Layers" icon="approve-check">
		Every primitive is tested at C# unit (snapshots + schema), TS unit (jsdom), and Playwright (browser). ~1,800 tests total.
	</Card>
	<Card title="Vertical Slices" icon="puzzle">
		Each component is a self-contained 7-file slice. Duplication over abstraction. Zero shared base classes.
	</Card>
	<Card title="Zero Runtime Changes" icon="rocket">
		Adding a new component requires only a new C# vertical slice. The 2KB JS runtime never changes.
	</Card>
</CardGrid>
